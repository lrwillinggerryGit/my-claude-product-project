# 随手记 · 财务管家记账本 💗

> 打开即记、记完即走，月底一眼看懂自己的钱。

一个**粉嫩可爱风**的个人记账 H5 应用，依据 [`001PRD/财务管家PRD-V1.0.md`](../001PRD/财务管家PRD-V1.0.md) 实现。
**纯前端、零依赖、零构建**——所有数据存在浏览器本地（localStorage），图表用 canvas 手绘，应用图标与 Excel 文件均由代码现场生成，**首次加载后可完全离线使用**。

## 🚀 运行方式

### 方式一：直接打开（最简单）
双击 [`index.html`](index.html) 用浏览器打开即可记账。
> 注意：`file://` 协议下 **Service Worker / 安装到主屏会自动跳过**（这两者需要 http/https），但记账等所有核心功能照常可用。

### 方式二：本地起静态服务（完整 PWA 体验，推荐）
在**仓库根目录**执行（已自带零依赖静态服务器）：

```bash
node .claude/static-server.cjs
# 然后浏览器打开 http://localhost:4321/claude-my-product-dist/
```

或用任意静态服务器指向本目录，例如：

```bash
npx serve claude-my-product/claude-my-product-dist
```

`localhost` 属于安全上下文，可启用 Service Worker 离线缓存、系统通知与「添加到主屏」。

## 💾 数据模型（localStorage 键：`suishouji_v1`）

```jsonc
{
  "ledgers": [                       // 多账本
    { "id": "default", "name": "默认账本", "icon": "💕",
      "records": [                   // 每个账本独立流水
        { "id": "...", "type": "expense|income", "amount": 35.8,
          "cat": "餐饮", "date": "2026-06-07", "note": "午饭", "ts": 1717... }
      ],
      "budgets": { "total": 6000, "cats": { "餐饮": 1500 } }  // 每个账本独立预算
    }
  ],
  "currentLedgerId": "default",
  "categories": {                    // 可编辑分类（预置带 preset:true）
    "expense": [ { "name": "餐饮", "icon": "🍜", "preset": true, "hidden": false } ],
    "income":  [ { "name": "工资", "icon": "💰", "preset": true } ]
  },
  "settings": {
    "lastCat": { "expense": "餐饮" },
    "lock": { "enabled": false, "hash": "" },     // PIN 加盐哈希（非明文）
    "reminder": { "enabled": false, "time": "21:00" },
    "largeFont": false
  }
}
```

- **自动迁移**：旧版（单账本 `records`/`budgets` + `customCats`）会在加载时自动升级为上面的结构，不丢数据。
- **备份文件**即该 JSON；「恢复备份」会再次归一化校验后导入。

## 🔧 技术亮点（零依赖实现）

- **图表**：饼图/趋势图用原生 `<canvas>` 手绘，含高 DPI 适配，无图表库。
- **应用图标**：`generate-icons.cjs` 用自写 PNG 编码器（CRC32 + zlib）绘制渐变爱心，并输出 maskable / apple-touch 等多规格。
- **Excel 导出**：浏览器端自写 **ZIP（存储法）+ 最小 OOXML**，导出含「明细 + 分类汇总」两个工作表，无需 SheetJS 等依赖。
- **离线**：Service Worker 缓存应用外壳（含字体）；字体非阻塞加载 + `display:swap`，首屏先用系统圆体兜底。

## ⚠️ 已知边界（受「纯前端无后端」限制）

- **每日提醒**仅在应用打开/后台运行时触发；网页**完全关闭时无法后台推送**（原生 App 能力）。
- **安全锁**为本地隐私门（PIN 以加盐哈希存储），非服务端强鉴权；忘记密码只能「清空数据重置」。
- PRD V2.0 的**原生 App / 小程序 / 多端云同步**需要原生工具链与后端服务，不在本纯前端版本范围内（多账本已用纯前端实现）。
