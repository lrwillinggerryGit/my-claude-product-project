## 1. 产品概述

### 1.1 背景

很多人有"想管好钱、但坚持不下来"的困扰：传统 Excel 记账门槛高、操作繁琐；银行/支付账单分散在多个 App，难以汇总；月底想复盘时根本说不清钱花在了哪里。用户真正需要的是一个**记录足够快、查看足够清、复盘足够直观**的轻量工具。

本产品面向有日常消费记录习惯（或希望养成该习惯）的个人用户，核心解决"**每天随手记一笔，月底看清钱去向**"这一刚需。

### 1.2 产品目标

- **核心目标**：让用户"记账这件事"的单次操作成本降到最低（目标：单笔记账 ≤ 10 秒）。
- **价值目标**：通过可视化统计与预算提醒，帮助用户建立对个人现金流的清晰认知，进而优化消费决策。
- **留存目标**：通过低门槛 + 正反馈（成就、连续记账天数、清晰报表），把"记账"培养成可持续的日常习惯。
- **（V1.0.1 新增）账号目标**：以最低门槛的"用户名 + 密码"为用户建立可识别身份，**为后续云备份与多端同步打基础**，但不能因登录环节牺牲"打开即记"的核心体验（首次注册/登录耗时目标 ≤ 30 秒）。

### 1.3 产品愿景

成为用户口袋里最顺手的"私人记账本"——打开即记、记完即走，月底一眼看懂自己的钱。

### 1.4 产品形态与平台范围

为了快速验证核心价值、降低首版开发成本，**第一版（V1.0）仅做 Web 网页端**，V1.0.1 保持同一形态与平台范围：

- **形态**：响应式 H5 网页应用，**以手机浏览器使用为主**（移动优先 / Mobile First），同时兼容 PC 浏览器访问。
- **访问方式**：用户通过浏览器打开网址即用，无需下载安装；支持"添加到主屏幕"获得近似 App 的启动体验（可选 PWA 能力）。
- **本期不做**：iOS / Android 原生 App、微信/支付宝小程序，均列入后续版本迭代。
- **设计取向**：交互布局以竖屏手机宽度为基准设计，PC 端做适配兼容即可，不单独投入做桌面级体验。

> 第一版的目标是用最小成本跑通"记账—查看—复盘"闭环并验证留存；待数据与需求验证后，再投入原生 App 与小程序，以获得离线、推送、入口便捷等更强的端能力。

## 2. 用户分析

### 2.1 目标用户

**用户范围**：面向 **18 岁以上**的所有人群（无年龄上限），覆盖学生、职场人、家庭/情侣到中老年用户。

**人群定位（主次）**：
- **主要群体（核心）**：**个人用户**，包括独立记账的个人，以及共同管理日常开支的**家庭 / 情侣**。产品的交互、分类、报表均以这部分用户为第一优先级设计。
- **次要群体（兼顾）**：**小微企业主 / 个体工商户**。他们收支规模不大、无需专业财务软件，但需要简单的收支两条线、公私区分与可导出的流水。本期不做专业财务功能，仅在分类、备注、导出等通用能力上予以兼顾。

| 用户类型 | 归属 | 特征 | 核心诉求 |
| --- | --- | --- | --- |
| 月光族 / 年轻职场人 | 个人（核心） | 收入稳定但存不下钱，消费随意 | 看清钱花在哪，控制冲动消费 |
| 学生 / 初入社会群体 | 个人（核心） | 可支配收入有限，需精打细算 | 低门槛记账、看清小额开支 |
| 家庭 / 情侣记账者 | 个人（核心） | 需共同管理家庭日常开支 | 分类清晰、可按月复盘、可设预算、可共享账本（远期） |
| 中老年 / 储蓄目标人群 | 个人（核心） | 重视存钱与稳健，为旅行、养老等攒钱 | 大字体易用、预算管理、目标进度可视化 |
| 自由职业者 | 个人 / 兼顾经营 | 收支不规律，需区分公私 | 收支两条线、可备注、可导出 |
| 小微企业主 / 个体工商户 | 次要（兼顾） | 经营规模小，无专业财务需求 | 简单收支记录、公私分账、流水导出对账 |

### 2.2 典型用户画像

> **小林，26 岁，互联网公司运营**
> 每月工资到账后没几天就"不知道花哪了"，想攒钱旅行却总是失败。试过几个记账 App，但都因为"记一笔要点好几下、还要选半天分类"而放弃。
> 她的诉求：**打开 App 两三步就能记完，月底能一眼看到这个月哪类花得最多。**

### 2.3 核心使用场景

1. **首次使用**：用户首次打开网页，按引导完成"用户名 + 密码"注册（≤ 30 秒），随后落到记账页直接开始记账。（V1.0.1 新增）
2. **再次访问**：用户已登录过，浏览器保留登录态，**默认直接进入记账页，无需重复登录**；登录态失效或主动退出后需重新登录。（V1.0.1 新增）
3. **即时记账**：在便利店买完东西，顺手掏出手机记一笔（金额 + 分类）。
4. **补记账**：晚上回家想起白天还有几笔没记，批量补录。
5. **月度复盘**：月底/月初查看上月消费报表，按分类查看占比与趋势。
6. **预算管控**：月初设定餐饮预算，月中收到"已用 80%"提醒。

## 3. 需求分析

### 3.1 用户痛点

- 记账操作步骤多、太麻烦 → **坚持不下来**。
- 账单分散、手动汇总累 → **看不清全貌**。
- 只有流水没有分析 → **不知道钱花在哪、该省哪里**。
- 没有预算约束 → **花超了也无感知**。
- **没有账号身份，换设备/清缓存后数据全部丢失，且无法在未来扩展云备份与多端同步**（V1.0.1 新增）。

### 3.2 需求优先级（MoSCoW）

| 优先级 | 需求 |
| --- | --- |
| Must（必须有） | 快速记账、收支分类、账单流水、月度统计、**用户账号体系（注册 / 登录 / 退出，V1.0.1 上调）** |
| Should（应该有） | 预算管理、超额提醒、连续记账激励、数据本地存储与备份 |
| Could（可以有） | 多账本、自定义分类、图表多维分析、数据导出 |
| Won't（本期不做） | 多人协作、银行卡自动同步、AI 智能记账、社区功能、**手机号/邮箱/第三方登录、找回密码、改密码（均列入后续版本）** |

> 说明：V1.0 时账号体系被列为风险/待确认问题；V1.0.1 决定**收敛为"用户名 + 密码"最小化方案**并上调至 Must。

## 4. 功能需求

### 4.1 功能结构图

```

│   └── 退出登录

│   ├── 快速记账（金额、收/支、分类、备注、日期）

│   └── 批量补记

│   ├── 单笔编辑 / 删除
│   └── 搜索与筛选

│   ├── 月度收支概览

    └── 设置（提醒、主题、安全锁）
```

### 4.2 核心功能详述

#### 4.2.1 快速记账（P0 核心）

**功能描述**：用户进入 App 后能以最少步骤完成一笔收支记录。

**字段定义**：

| 字段 | 是否必填 | 说明 | 默认值 |
| --- | --- | --- | --- |
| 金额 | 必填 | 数字键盘输入，支持小数点后两位 | 无 |
| 收支类型 | 必填 | 支出 / 收入，默认"支出" | 支出 |
| 分类 | 必填 | 点选分类图标 | 上次使用的分类 |
| 日期 | 必填 | 默认当天，可改 | 当天 |
| 备注 | 选填 | 文本输入，≤ 50 字 | 空 |

**交互要求**：
- 打开 App 默认落在记账页，金额输入框自动聚焦、数字键盘自动弹出。
- 分类以图标网格平铺展示，单击即选中，无需二级菜单。
- "保存"后给出轻量成功反馈（如金额飞入动画 + Toast），并清空表单等待下一笔。
- 单笔记账理想路径：**输入金额 → 点分类 → 保存**，3 步完成，目标耗时 ≤ 10 秒。
- **（V1.0.1）若用户未登录，访问记账页将先跳转至登录页**；登录成功后回到记账页继续操作。

**异常处理**：
- 金额为 0 或为空时，"保存"按钮置灰不可点。
- 金额超过单笔上限（如 1,000,000）时给出提示。

#### 4.2.2 收支分类

**功能描述**：提供系统预置分类，并支持用户自定义。

- **预置支出分类**：餐饮、交通、购物、居住、娱乐、医疗、教育、人情、通讯、其他。
- **预置收入分类**：工资、奖金、理财、兼职、红包、其他。
- **自定义分类**：用户可新增分类（图标 + 名称）、隐藏不用的分类、调整排序。
- 删除分类时若该分类下已有账单，需提示"历史账单将归入'其他'"并二次确认。
- **（V1.0.1）分类配置以"账号"为粒度**：A 账号定义的自定义分类不会出现在 B 账号下。

#### 4.2.3 账单流水

**功能描述**：以时间维度展示所有收支记录。

- 默认按"月"分组展示，顶部显示当月总收入、总支出、结余。
- 列表项展示：分类图标、分类名、备注、日期、金额（支出为红色/收入为绿色）。
- 支持按日/周/月切换视图。
- 单击列表项进入详情，可编辑或删除。
- 支持按关键词（备注）、分类、金额区间、日期范围筛选搜索。
- **（V1.0.1）账单数据严格按账号隔离**：只展示当前登录用户的记录。

#### 4.2.4 统计分析

**功能描述**：将流水转化为可视化洞察。

- **收支概览**：当月收入、支出、结余的数字卡片，并与上月环比。
- **分类占比**：饼图展示各分类支出占比，点击可下钻查看该分类明细。
- **趋势分析**：折线/柱状图展示近 6 个月或近 30 天的收支趋势。
- 支持切换统计周期（本月、上月、本年、自定义区间）。

#### 4.2.5 预算管理（P1）

**功能描述**：帮助用户设定花费上限并主动管控。

- 支持设置**月度总预算**与**分类预算**。
- 首页/统计页以进度条展示"已用 / 预算"，并按比例变色（绿 → 黄 → 红）。
- 触达阈值时推送提醒：达到 80%、100%、超额时分别提醒。
- 预算可设为每月自动循环。

#### 4.2.6 习惯激励（P1）

**功能描述**：通过正反馈提升记账留存。

- 连续记账天数统计（连续打卡）。
- 每日记账提醒（可设定时间，默认晚 21:00）。
- 里程碑成就（如"连续记账 7 天""累计记账 100 笔"）。
- **（V1.0.1）打卡、成就、提醒设置全部以账号为单位**，切换账号后各自独立。

#### 4.2.7 数据安全与备份（P1）

- **本地存储优先**：数据默认存储在本地，保证离线可用。
- **备份/恢复**：支持导出备份文件、从备份恢复；可选云端备份（**V1.0.1 仍仅作为"账号体系打基础"，云备份能力本期不实现，列入 V1.2**）。
- **数据导出**：支持导出 CSV / Excel，便于二次分析。
- **隐私安全**：支持手势密码 / 指纹 / 面容锁。

#### 4.2.8 用户账号体系（V1.0.1 新增，P0）

**功能描述**：本期引入最小化的账号能力，让用户拥有一个可识别身份，并以账号为粒度隔离数据。**仅支持"用户名 + 密码"一种方式**。

##### 4.2.8.1 注册

**字段定义**：

| 字段 | 是否必填 | 规则 | 错误提示示例 |
| --- | --- | --- | --- |
| 用户名 | 必填 | 长度 4–20，仅允许**字母 / 数字 / 下划线**，必须以字母或数字开头；**全局唯一**，不区分大小写存储与校验 | "用户名格式不符（4–20 位字母/数字/下划线）"<br>"该用户名已被注册" |
| 密码 | 必填 | 长度 6–20，必须**同时包含字母与数字**，不允许空格 | "密码需 6–20 位，且同时包含字母与数字" |
| 确认密码 | 必填 | 必须与"密码"完全一致 | "两次输入的密码不一致" |

**交互要求**：
- 注册页提供"用户名 / 密码 / 确认密码"三个输入框 + 显示/隐藏密码切换 + "注册"按钮 + "已有账号？去登录"链接。
- 输入框失焦时即时校验格式；"注册"按钮在所有字段满足规则前置灰。
- 注册成功后**自动登录**并落到记账页，**不需要让用户再手动登录一次**。
- 注册成功时同步为该账号初始化默认账本、预置分类及空预算。

**安全要求**：
- 密码**严禁明文存储**：使用**加盐哈希**保存（建议 bcrypt 或 PBKDF2，迭代次数符合当年安全实践），数据库/本地存储中只保留 salt + hash。
- 注册接口需有**频次限制**（同一 IP / 设备 1 分钟内 ≤ 5 次），防止脚本批量注册。
- 错误提示**不区分**"用户名不存在"与"密码错误"，统一为"用户名或密码错误"，避免用户名枚举（适用于登录场景，下同）。

##### 4.2.8.2 登录

**字段定义**：

| 字段 | 是否必填 | 规则 |
| --- | --- | --- |
| 用户名 | 必填 | 与注册时大小写不敏感匹配 |
| 密码 | 必填 | 6–20 位 |

**交互要求**：
- 登录页提供"用户名 / 密码"输入框 + 显示/隐藏密码切换 + "登录"按钮 + "还没账号？去注册"链接 + "记住我"勾选框（默认勾选）。
- 登录失败统一提示"用户名或密码错误"。
- **同一账号连续 5 次密码错误后锁定 15 分钟**，期间任何来源的登录请求均直接拒绝，提示"账号已锁定，请 X 分钟后再试"。
- 登录成功后落到记账页。

##### 4.2.8.3 登录态保持

- 登录成功后服务器/本地下发**会话凭证（token）**：
  - 勾选"记住我"：有效期 30 天，**到期前用户每次访问自动续期**。
  - 未勾选："仅本次会话"，关闭浏览器或 8 小时后失效。
- 任何受保护页面（记账、账单、统计、预算、我的）在请求时校验凭证；**凭证缺失/失效时自动跳转登录页，并在登录成功后回到原跳转目标**。
- 凭证**不放在 URL 中**；前端存储位置使用 HttpOnly Cookie（条件允许时）或 localStorage（次选），并防范 XSS。

##### 4.2.8.4 退出登录

- 入口：「我的」页面底部"退出登录"按钮。
- 点击需**二次确认**："退出后本地数据仍保留，再次登录即可看到。确认退出？"。
- 确认后：清除会话凭证 → 跳转登录页 → **本地业务数据保留不删除**。

##### 4.2.8.5 数据归属与隔离

- 所有业务数据（账单、分类、预算、提醒设置、成就、连续打卡天数等）**以 `userId` 为外键**归属到某个账号。
- **同一浏览器多账号切换**：A 退出 → B 登录 → 只展示 B 的数据；B 退出 → A 重新登录 → 看到 A 原来的全部数据，**互不串扰、互不可见**。
- V1.0.1 仍为前端纯本地存储形态，数据按账号分键命名（如 `suishouji_v1_user_<userId>`）；当未来引入云端时，迁移路径不变。

##### 4.2.8.6 异常处理与边界

| 场景 | 系统行为 |
| --- | --- |
| 网络断开时点"注册/登录" | Toast 提示"网络不可用，请检查后重试"，按钮恢复可点状态 |
| 同一用户名并发注册（极端） | 后端唯一索引兜底，先到先得；后到者收到"该用户名已被注册" |
| 用户名空白或仅含空格 | 视为格式错误，前端阻断 |
| 用户已登录又再次访问"登录/注册页" | 直接跳转记账页，避免重复登录 |
| 凭证被篡改或解析失败 | 视为未登录，跳转登录页 |

##### 4.2.8.7 本期不做（明确剔除，避免范围蔓延）

- 找回密码 / 修改密码 / 注销账号（列入 V1.2 与账号体系一同打磨）
- 手机号 / 邮箱 / 第三方（微信、Apple ID 等）登录与绑定
- 二次验证（短信验证码 / TOTP）
- 多设备登录互踢、登录设备管理
- 管理员后台

### 4.4 视觉设计规范（粉嫩可爱风）

**设计风格**：整体采用**粉嫩、可爱、温柔治愈**的视觉基调，弱化"理财=严肃数字"的距离感，让记账变得轻松有亲和力。圆角、柔和阴影、留白充足，搭配少量可爱插画/表情元素。

**主色板**：

| 角色 | 颜色名 | 色值（参考） | 用途 |
| --- | --- | --- | --- |
| 主色 | 樱花粉 | `#FF9EC4` | 主按钮、导航高亮、品牌色 |
| 主色浅 | 蜜桃粉 | `#FFD6E7` | 选中态背景、卡片高亮、进度条底 |
| 辅助色 | 奶杏橙 | `#FFC8A2` | 次要按钮、标签点缀 |
| 点缀色 | 薄荷绿 | `#A8E6CF` | 收入正向元素、成就徽章 |
| 背景色 | 奶油白 | `#FFF8FB` | 页面背景（带极浅粉调） |
| 文字主色 | 暖灰 | `#5C5560` | 正文（避免纯黑，更柔和） |

**收支语义色**（兼顾色弱）：
- 支出：玫瑰红 `#FF6F91`，并以"-"前缀强化。
- 收入：薄荷绿 `#3DBE8B`，并以"+"前缀强化。

**风格细节**：
- 圆角统一偏大（卡片 16px、按钮 24px 胶囊形），营造柔软感。
- 阴影用粉调柔光（如 `rgba(255,158,196,0.25)`），不用生硬黑色投影。
- 字体偏圆润（如苹方/思源黑体 Medium，可点缀圆体数字）。
- 适度使用可爱元素：记账成功的小动画、连续打卡的萌系徽章、空状态的治愈插画。
- 保证对比度与可读性：粉色背景上的文字需达到 WCAG AA 对比度，关键数字不因配色牺牲清晰度。

**（V1.0.1 新增）注册/登录页视觉**：
- 顶部放置 1 个圆角粉色品牌头图（可放小猪存钱罐 / 小熊抱钱袋等治愈插画占位），下方为表单卡片。
- 输入框使用奶油白底 + 樱花粉聚焦描边，错误态使用玫瑰红文案 + 柔和抖动动画，避免严厉感。
- 主按钮（注册 / 登录）使用樱花粉胶囊按钮，禁用态降饱和到 40%。
- 文案语气保持治愈："欢迎回来呀～""第一次来？三十秒就能开始记账啦"。

## 5. 业务流程

### 5.1 注册主流程（V1.0.1 新增）

```
首次进入 → 落地登录页
  → 点"还没账号？去注册" → 进入注册页
  → 输入 用户名 / 密码 / 确认密码
  → 前端实时校验格式（不通过则按钮置灰 + 局部错误提示）
  → 点击"注册"
  → 服务端校验唯一性 + 频控
      ├── 通过 → 写入用户表（密码加盐哈希）→ 自动登录（下发 token）→ 初始化默认账本/分类/空预算 → 跳转记账页
      └── 不通过 → 在对应字段给出错误（如"该用户名已被注册"），表单保留输入
```

### 5.2 登录主流程（V1.0.1 新增）

```
访问任意受保护页面（默认记账页）
  → 检查本地是否有有效 token
      ├── 有效 → 直接进入目标页
      └── 无 / 失效 → 跳转登录页（记住原 url）
  → 输入 用户名 / 密码 → 点"登录"
  → 服务端校验
      ├── 成功 → 下发 token（按"记住我"决定有效期）→ 回到原 url（默认记账页）
      ├── 失败 → 累计错误次数，提示"用户名或密码错误"
      └── 达到 5 次 → 锁定 15 分钟，提示"账号已锁定，请 X 分钟后再试"
```

### 5.3 退出登录流程（V1.0.1 新增）

```
「我的」页 → 点"退出登录"
  → 二次确认弹窗 ("退出后本地数据仍保留")
      ├── 取消 → 留在原页
      └── 确认 → 清除 token / 会话信息 → 跳转登录页（本地业务数据保留）
```

### 5.4 记账主流程

```
（已登录用户）打开 App → 默认进入记账页（金额框聚焦）
  → 输入金额
  → 选择收/支类型（默认支出）
  → 点选分类
  → （可选）填备注 / 改日期
  → 点击保存
  → 校验金额是否合法
      ├── 合法 → 写入本地数据库（绑定当前 userId） → 成功反馈 → 清空表单
      └── 不合法 → 提示错误，停留当前页

（未登录用户）打开 App → 跳转登录页 → 完成登录 → 回到记账页继续上述流程
```

### 5.5 预算提醒流程

```
用户设定分类预算
  → 每笔记账后实时累计该分类当月支出
  → 计算 已用/预算 比例
      ├── < 80%  → 仅更新进度条
      ├── ≥ 80%  → 推送"接近预算"提醒
      └── ≥ 100% → 推送"已超预算"提醒（高亮红色）
```
