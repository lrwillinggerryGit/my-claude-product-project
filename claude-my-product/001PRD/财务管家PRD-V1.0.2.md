## 1. 产品概述

### 1.1 背景

很多人有"想管好钱、但坚持不下来"的困扰：传统 Excel 记账门槛高、操作繁琐；银行/支付账单分散在多个 App，难以汇总；月底想复盘时根本说不清钱花在了哪里。用户真正需要的是一个**记录足够快、查看足够清、复盘足够直观**的轻量工具。

本产品面向有日常消费记录习惯（或希望养成该习惯）的个人用户，核心解决"**每天随手记一笔，月底看清钱去向**"这一刚需。

### 1.2 产品目标

- **核心目标**：让用户"记账这件事"的单次操作成本降到最低（目标：单笔记账 ≤ 10 秒）。
- **价值目标**：通过可视化统计与预算提醒，帮助用户建立对个人现金流的清晰认知，进而优化消费决策。
- **留存目标**：通过低门槛 + 正反馈（成就、连续记账天数、清晰报表），把"记账"培养成可持续的日常习惯。
- **账号目标**：以最低门槛的"用户名 + 密码"为用户建立可识别身份，**为后续云备份与多端同步打基础**，但不能因登录环节牺牲"打开即记"的核心体验（首次注册/登录耗时目标 ≤ 30 秒）。

### 1.3 产品愿景

成为用户口袋里最顺手的"私人记账本"——打开即记、记完即走，月底一眼看懂自己的钱。**（V1.0.2）品牌定名"财务管家"，品牌 slogan"管理您的每一分钱"，以粉嫩治愈的视觉语言弱化理财的严肃感。**

### 1.4 产品形态与平台范围

第一版仍为 Web 网页端：

- **形态**：Web 网页应用。**（V1.0.2 实现现状）** 登录 / 注册页为**移动优先的响应式卡片布局**；而首页、交易、统计、设置、分类、快速记账等核心功能页当前采用**桌面宽屏 + 左侧固定侧边栏导航**的布局形态。
- **访问方式**：用户通过浏览器打开网址即用，无需下载安装；支持"添加到主屏幕"获得近似 App 的启动体验（可选 PWA 能力）。
- **本期不做**：iOS / Android 原生 App、微信/支付宝小程序，均列入后续版本迭代。
- **设计取向**：**（V1.0.2 修订）** 当前原型以桌面宽屏为主要呈现形态，移动端适配作为并行推进项。后续需在"移动优先"与"桌面宽屏"之间明确主基准（见风险 #8）。

> 第一版的目标是用最小成本跑通"记账—查看—复盘"闭环并验证留存；待数据与需求验证后，再投入原生 App 与小程序，以获得离线、推送、入口便捷等更强的端能力。

## 2. 用户分析

### 2.1 目标用户

**用户范围**：面向 **18 岁以上**的所有人群（无年龄上限），覆盖学生、职场人、家庭/情侣到中老年用户。

**人群定位（主次）**：
- **主要群体（核心）**：**个人用户**，包括独立记账的个人，以及共同管理日常开支的**家庭 / 情侣**。产品的交互、分类、报表均以这部分用户为第一优先级设计。
- **次要群体（兼顾）**：**小微企业主 / 个体工商户**。他们收支规模不大、无需专业财务软件，但需要简单的收支两条线、公私区分与可导出的流水。本期不做专业财务功能，仅在分类、备注、导出等通用能力上予以兼顾。

| 用户类型 | 归属 | 特征 | 核心诉求 |
| --- | --- | --- | --- |
| 月光族 / 年轻职场人 | 个人（核心） | 收入稳定但存不下钱，消费随意 | 看清钱花在哪，控制冲动消费 |
| 学生 / 初入社会群体 | 个人（核心） | 可支配收入有限，需精打细算 | 低门槛记账、看清小额开支 |
| 家庭 / 情侣记账者 | 个人（核心） | 需共同管理家庭日常开支 | 分类清晰、可按月复盘、可设预算、可共享账本（远期） |
| 中老年 / 储蓄目标人群 | 个人（核心） | 重视存钱与稳健，为旅行、养老等攒钱 | 大字体易用、预算管理、目标进度可视化 |
| 自由职业者 | 个人 / 兼顾经营 | 收支不规律，需区分公私 | 收支两条线、可备注、可导出 |
| 小微企业主 / 个体工商户 | 次要（兼顾） | 经营规模小，无专业财务需求 | 简单收支记录、公私分账、流水导出对账 |

### 2.2 典型用户画像

> **小林，26 岁，互联网公司运营**
> 每月工资到账后没几天就"不知道花哪了"，想攒钱旅行却总是失败。试过几个记账 App，但都因为"记一笔要点好几下、还要选半天分类"而放弃。
> 她的诉求：**打开 App 两三步就能记完，月底能一眼看到这个月哪类花得最多。**

### 2.3 核心使用场景

1. **首次使用**：用户首次打开网页，按引导完成"用户名 + 密码"注册（≤ 30 秒），随后落到记账/首页直接开始记账。
2. **再次访问**：用户已登录过，浏览器保留登录态，**默认直接进入首页/记账页，无需重复登录**；登录态失效或主动退出后需重新登录。
3. **即时记账**：在便利店买完东西，顺手掏出手机记一笔（金额 + 分类 + **支付账户**）。
4. **补记账**：晚上回家想起白天还有几笔没记，批量补录。
5. **月度复盘**：月底/月初查看上月消费报表，按分类查看占比与趋势。
6. **预算管控**：月初设定餐饮预算，月中收到"已用 80%"提醒。

## 3. 需求分析

### 3.1 用户痛点

- 记账操作步骤多、太麻烦 → **坚持不下来**。
- 账单分散、手动汇总累 → **看不清全貌**。
- 只有流水没有分析 → **不知道钱花在哪、该省哪里**。
- 没有预算约束 → **花超了也无感知**。
- 没有账号身份，换设备/清缓存后数据全部丢失，且无法在未来扩展云备份与多端同步。

### 3.2 需求优先级（MoSCoW）

| 优先级 | 需求 |
| --- | --- |
| Must（必须有） | 快速记账、收支分类、账单流水、月度统计、用户账号体系（注册 / 登录 / 退出）、**（V1.0.2）支付账户管理** |
| Should（应该有） | 预算管理、超额提醒、连续记账激励、数据本地存储与备份 |
| Could（可以有） | 多账本、自定义分类、图表多维分析、数据导出 |
| Won't（本期不做） | 多人协作、银行卡自动同步、AI 智能记账、社区功能、手机号/邮箱/第三方登录、找回密码、改密码（均列入后续版本） |

> **（V1.0.2 说明）** 前端原型的登录页已放置"忘记密码"、设置页已放置"修改密码"、交易页已放置"导出报表"入口，但这些**均属本期不做项，当前仅为界面占位**，不代表本期功能范围。

### 4.2 核心功能详述

#### 4.2.1 快速记账（P0 核心）

**功能描述**：用户进入 App 后能以最少步骤完成一笔收支记录。

**字段定义（V1.0.2 更新）**：

| 字段 | 是否必填 | 说明 | 默认值 |
| --- | --- | --- | --- |
| 金额 | 必填 | 数字键盘输入，支持小数点后两位 | 无 |
| 收支类型 | 必填 | 支出 / 收入，默认"支出" | 支出 |
| 分类 | 必填 | 点选分类图标（图标网格平铺，超 10 项进入"更多"弹层） | 切换类型后默认选第一个分类 |
| **支付账户** | **选填（V1.0.2 新增）** | **点选支付账户（微信/支付宝/银行卡等）** | **默认第一个账户** |
| 日期 | 必填 | 默认当天，可改 | 当天 |
| **商户 / 标题** | **选填（V1.0.2 新增）** | **交易对象名称，如"全食超市""星巴克"** | 空 |
| 备注 | 选填 | 文本输入，≤ 50 字 | 空 |

**交互要求**：
- 打开 App 默认落在首页/记账页，金额输入框自动聚焦。
- 分类以图标网格平铺展示，单击即选中，无需二级菜单；**（V1.0.2）当分类超过 10 个（5 列 × 2 行）时，末位显示"更多"入口，点击弹出全部分类选择弹层**。
- **（V1.0.2）支付账户以卡片列表呈现，单选，选中项以樱花粉描边 + 圆点标记。**
- "保存"后给出轻量成功反馈（如按钮变绿"✓ 已保存"），随后跳转交易列表。
- 单笔记账理想路径：**输入金额 → 点分类 → 保存**，3 步完成，目标耗时 ≤ 10 秒。
- 若用户未登录，访问记账页将先跳转至登录页；登录成功后回到记账页继续操作。

**异常处理**：
- 金额为 0 或为空时，"保存"按钮置灰不可点。
- 金额超过单笔上限（如 1,000,000）时给出提示。

#### 4.2.2 收支分类

**功能描述**：提供系统预置分类，并支持用户自定义。

- **（V1.0.2 更新）预置支出分类**：餐饮 🍜、购物 🛒、交通 🚌、居家 🏠、房租 🏡、娱乐 🎬、医疗 💊、教育 📚、通讯 📱、人情 💝。
- **（V1.0.2 更新）预置收入分类**：工资 💰、劳务 💼、外快 🧧、投资 📈、红包 🎁。
- **自定义分类**：用户可新增分类（emoji 图标 + 名称，名称 ≤ 8 字）、编辑、删除、隐藏不用的分类、调整排序。**（V1.0.2）分类编辑弹层提供 emoji 图标选择器（内置约 48 个可爱表情）与实时预览。**
- 删除分类时需**二次确认**；若该分类下已有账单，历史账单归入"其他"。
- 分类配置以"账号"为粒度：A 账号定义的自定义分类不会出现在 B 账号下。

> **（V1.0.2 实现备注）** 当前前端原型的分类数据存于独立键 `suishouji_categories_v1`，尚未按账号分键，属实现待对齐项（见风险 #9）。

#### 4.2.3 账单 / 交易流水

**功能描述**：以时间维度展示所有收支记录。

- **（V1.0.2）默认按"日"分组展示**（如"今天""10月24日 周四"），每组顶部显示当日收/支合计。
- 顶部展示本月总收入、总支出卡片，并与上月环比。
- 列表项展示：分类图标、商户/标题、时间 · 支付账户、金额（支出为红色、收入为绿色）。
- 单击列表项进入详情/编辑，可编辑或删除。
- 支持按关键词（商户/备注）、**分类、支付账户**、日期范围筛选搜索。
- 账单数据严格按账号隔离：只展示当前登录用户的记录。

#### 4.2.4 首页 / 财务概览（V1.0.2 明确）

**功能描述**：登录后的默认落地页，一屏概览本月财务健康度。

- **收支概览卡片**：本月总收入、总支出，各自附"上月同期"对比数字。
- **最近交易明细**：展示最近数笔交易（分类图标 + 商户 + 日期·账户 + 带符号金额），提供"查看全部"跳转交易页。
- **支出分类占比**：进度条列表 + 环形图，展示各分类占比与本月支出总计。

#### 4.2.5 统计分析

**功能描述**：将流水转化为可视化洞察。

- **收支概览**：当月收入、支出、结余（或支出大头）的数字卡片，并与上月环比（↑/↓ 百分比）。
- **分类占比**：环形图展示各分类支出占比，配图例与百分比。
- **趋势分析**：折线/面积图展示支出趋势，可叠加预算对比线；**（V1.0.2）周期切换支持"按月 / 按季 / 按年"**。
- **支出明细**：分类支出排行列表（金额 + 占比进度条）。

#### 4.2.6 预算管理（P1）

**功能描述**：帮助用户设定花费上限并主动管控。

- 支持设置**月度总预算**与**分类预算**。
- 首页/统计页以进度条展示"已用 / 预算"，并按比例变色（绿 → 黄 → 红）。
- 触达阈值时推送提醒：达到 80%、100%、超额时分别提醒。
- 预算可设为每月自动循环。

#### 4.2.7 习惯激励（P1）

**功能描述**：通过正反馈提升记账留存。

- 连续记账天数统计（连续打卡）。
- 每日记账提醒（可设定时间，默认晚 21:00）。
- 里程碑成就（如"连续记账 7 天""累计记账 100 笔"）。
- 打卡、成就、提醒设置全部以账号为单位，切换账号后各自独立。

#### 4.2.8 数据安全与备份（P1）

- **本地存储优先**：数据默认存储在本地，保证离线可用。
- **备份/恢复**：支持导出备份文件、从备份恢复；可选云端备份（云备份能力本期不实现，列入 V1.2）。
- **数据导出**：支持导出 CSV / Excel，便于二次分析。**（V1.0.2）交易页已放置"导出报表"入口占位，功能列入后续版本。**
- **隐私安全**：支持手势密码 / 指纹 / 面容锁。**（V1.0.2）设置页已提供"指纹/面容解锁"开关。**

#### 4.2.9 个人资料与支付账户管理（V1.0.2 新增，P1）

**个人资料（设置页）**：
- **头像**：默认为可爱 emoji（如 🐱），支持上传本地图片（仅前端预览，≤ 5MB，需图片类型）。
- **昵称**：文本，默认展示用户昵称。
- **性别**：女 / 男 / 保密。
- **个性签名**：文本，≤ 200 字。

**支付账户管理（设置页 / 记账页）**：
- 支持管理多个支付账户：微信支付 💬、支付宝 💳、银行卡 🏦（可含尾号，如"尾号 8888"）、信用卡、现金等。
- 记账时可将该笔交易归属到某个支付账户。
- 账户以卡片形式展示绑定状态，支持编辑。

#### 4.2.10 用户账号体系（P0）

**功能描述**：本期引入最小化的账号能力，让用户拥有一个可识别身份，并以账号为粒度隔离数据。**仅支持"用户名 + 密码"一种方式。**

##### 4.2.10.1 注册

**字段定义**：

| 字段 | 是否必填 | 规则 | 错误提示示例 |
| --- | --- | --- | --- |
| 用户名 | 必填 | 长度 4–20，仅允许**字母 / 数字 / 下划线**，必须以字母或数字开头；**全局唯一**，不区分大小写存储与校验（前端正则 `^[a-zA-Z0-9][a-zA-Z0-9_]{3,19}$`） | "用户名格式不符（4–20 位字母/数字/下划线，须以字母或数字开头）"<br>"该用户名已被注册" |
| 密码 | 必填 | 长度 6–20，必须**同时包含字母与数字**，不允许空格（前端正则 `^(?=.*[a-zA-Z])(?=.*\d)[^\s]{6,20}$`） | "密码需 6–20 位，且同时包含字母与数字" |
| 确认密码 | 必填 | 必须与"密码"完全一致 | "两次输入的密码不一致" |
| **同意协议** | **必填** | **需勾选"我已阅读并同意服务协议与隐私政策"方可注册** | — |

**交互要求**：
- 注册页提供"用户名 / 密码 / 确认密码"三个输入框 + 显示/隐藏密码切换 + 同意协议勾选框 + "注册"按钮 + "已有账号？去登录"链接。
- 输入框输入时即时校验格式；"注册"按钮在所有字段满足规则**且勾选协议**前置灰。
- 注册成功后**自动登录**并落到首页/记账页，**不需要让用户再手动登录一次**。
- 注册成功时同步为该账号初始化默认账本、预置分类及空预算。

**安全要求**：
- 密码**严禁明文存储**：使用**加盐哈希**保存（建议 bcrypt 或 PBKDF2），数据库/本地存储中只保留 salt + hash。
- 注册接口需有**频次限制**（同一 IP / 设备 1 分钟内 ≤ 5 次），防止脚本批量注册。
- 错误提示**不区分**"用户名不存在"与"密码错误"，统一为"用户名或密码错误"，避免用户名枚举（适用于登录场景）。

##### 4.2.10.2 登录

**字段定义**：

| 字段 | 是否必填 | 规则 |
| --- | --- | --- |
| 用户名 | 必填 | 与注册时大小写不敏感匹配 |
| 密码 | 必填 | 6–20 位 |

**交互要求**：
- 登录页提供"用户名 / 密码"输入框 + 显示/隐藏密码切换 + "登录"按钮 + "还没账号？去注册"链接 + "记住我（30 天内免登录）"勾选框（默认勾选）。
- 登录失败统一提示"用户名或密码错误"。
- **同一账号连续 5 次密码错误后锁定 15 分钟**，期间任何来源的登录请求均直接拒绝，提示"账号已锁定，请 X 分钟后再试"。
- 登录成功后落到首页/记账页。

##### 4.2.10.3 登录态保持

- 登录成功后服务器/本地下发**会话凭证（token）**：
  - 勾选"记住我"：有效期 30 天，**到期前用户每次访问自动续期**。
  - 未勾选："仅本次会话"，关闭浏览器或 8 小时后失效。
- 任何受保护页面在请求时校验凭证；**凭证缺失/失效时自动跳转登录页，并在登录成功后回到原跳转目标**。
- 凭证**不放在 URL 中**；前端存储位置使用 HttpOnly Cookie（条件允许时）或 localStorage（次选），并防范 XSS。

##### 4.2.10.4 退出登录

- 入口：「设置」页底部"退出登录"按钮。
- 点击需**二次确认**："退出后本地数据仍保留，再次登录即可看到。确认退出？"。
- 确认后：清除会话凭证 → 跳转登录页 → **本地业务数据保留不删除**。

##### 4.2.10.5 数据归属与隔离

- 所有业务数据（账单、分类、预算、支付账户、提醒设置、成就、连续打卡天数等）**以 `userId` 为外键**归属到某个账号。
- **同一浏览器多账号切换**：A 退出 → B 登录 → 只展示 B 的数据；B 退出 → A 重新登录 → 看到 A 原来的全部数据，**互不串扰、互不可见**。
- 数据按账号分键命名（如 `suishouji_v1_user_<userId>`）；当未来引入云端时，迁移路径不变。

##### 4.2.10.6 异常处理与边界

| 场景 | 系统行为 |
| --- | --- |
| 网络断开时点"注册/登录" | Toast 提示"网络不可用，请检查后重试"，按钮恢复可点状态 |
| 同一用户名并发注册（极端） | 后端唯一索引兜底，先到先得；后到者收到"该用户名已被注册" |
| 用户名空白或仅含空格 | 视为格式错误，前端阻断 |
| 用户已登录又再次访问"登录/注册页" | 直接跳转首页/记账页，避免重复登录 |
| 凭证被篡改或解析失败 | 视为未登录，跳转登录页 |

##### 4.2.10.7 本期不做（明确剔除，避免范围蔓延）

- 找回密码 / 修改密码 / 注销账号（列入 V1.2；**前端登录页"忘记密码"、设置页"修改密码"仅为占位入口**）
- 手机号 / 邮箱 / 第三方（微信、Apple ID 等）登录与绑定
- 二次验证（短信验证码 / TOTP）
- 多设备登录互踢、登录设备管理
- 管理员后台

## 5. 业务流程

### 5.1 注册主流程

```
首次进入 → 落地登录页
  → 点"还没账号？去注册" → 进入注册页
  → 输入 用户名 / 密码 / 确认密码 + 勾选同意协议
  → 前端实时校验格式（不通过则按钮置灰 + 局部错误提示）
  → 点击"注册"
  → 服务端校验唯一性 + 频控
      ├── 通过 → 写入用户表（密码加盐哈希）→ 自动登录（下发 token）→ 初始化默认账本/分类/支付账户/空预算 → 跳转首页/记账页
      └── 不通过 → 在对应字段给出错误（如"该用户名已被注册"），表单保留输入
```

### 5.2 登录主流程

```
访问任意受保护页面（默认首页/记账页）
  → 检查本地是否有有效 token
      ├── 有效 → 直接进入目标页
      └── 无 / 失效 → 跳转登录页（记住原 url）
  → 输入 用户名 / 密码 → 点"登录"
  → 服务端校验
      ├── 成功 → 下发 token（按"记住我"决定有效期）→ 回到原 url（默认首页/记账页）
      ├── 失败 → 累计错误次数，提示"用户名或密码错误"
      └── 达到 5 次 → 锁定 15 分钟，提示"账号已锁定，请 X 分钟后再试"
```

### 5.3 退出登录流程

```
「设置」页 → 点"退出登录"
  → 二次确认弹窗 ("退出后本地数据仍保留")
      ├── 取消 → 留在原页
      └── 确认 → 清除 token / 会话信息 → 跳转登录页（本地业务数据保留）
```

### 5.4 记账主流程（V1.0.2 更新）

```
（已登录用户）打开 App → 默认进入首页/记账页（金额框聚焦）
  → 输入金额
  → 选择收/支类型（默认支出）
  → 点选分类（超 10 项时可从"更多"弹层选择）
  → 选择支付账户（默认第一个）
  → （可选）填商户/标题、备注 / 改日期
  → 点击保存
  → 校验金额是否合法（0 < 金额 ≤ 1,000,000）
      ├── 合法 → 写入本地数据库（绑定当前 userId） → 成功反馈 → 跳转交易列表
      └── 不合法 → 提示错误 / 保存按钮置灰，停留当前页

（未登录用户）打开 App → 跳转登录页 → 完成登录 → 回到记账页继续上述流程
```

### 5.5 预算提醒流程

```
用户设定分类预算
  → 每笔记账后实时累计该分类当月支出
  → 计算 已用/预算 比例
      ├── < 80%  → 仅更新进度条
      ├── ≥ 80%  → 推送"接近预算"提醒
      └── ≥ 100% → 推送"已超预算"提醒（高亮红色）
```
