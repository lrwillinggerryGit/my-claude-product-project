## 1. 产品概述

### 1.1 背景

很多人有"想管好钱、但坚持不下来"的困扰：传统 Excel 记账门槛高、操作繁琐；银行/支付账单分散在多个 App，难以汇总；月底想复盘时根本说不清钱花在了哪里。用户真正需要的是一个**记录足够快、查看足够清、复盘足够直观**的轻量工具。

本产品面向有日常消费记录习惯（或希望养成该习惯）的个人用户，核心解决"**每天随手记一笔，月底看清钱去向**"这一刚需。

### 1.2 产品目标

- **核心目标**：让用户"记账这件事"的单次操作成本降到最低（目标：单笔记账 ≤ 10 秒）。
- **价值目标**：通过可视化统计与预算提醒，帮助用户建立对个人现金流的清晰认知，进而优化消费决策。
- **留存目标**：通过低门槛 + 正反馈（成就、连续记账天数、清晰报表），把"记账"培养成可持续的日常习惯。

### 1.3 产品愿景

成为用户口袋里最顺手的"私人记账本"——打开即记、记完即走，月底一眼看懂自己的钱。

### 1.4 产品形态与平台范围

为了快速验证核心价值、降低首版开发成本，**第一版（V1.0）仅做 Web 网页端**，形态与平台范围如下：

- **形态**：响应式 H5 网页应用，**以手机浏览器使用为主**（移动优先 / Mobile First），同时兼容 PC 浏览器访问。
- **访问方式**：用户通过浏览器打开网址即用，无需下载安装；支持"添加到主屏幕"获得近似 App 的启动体验（可选 PWA 能力）。
- **本期不做**：iOS / Android 原生 App、微信/支付宝小程序，均列入后续版本迭代。
- **设计取向**：交互布局以竖屏手机宽度为基准设计，PC 端做适配兼容即可，不单独投入做桌面级体验。

> 第一版的目标是用最小成本跑通"记账—查看—复盘"闭环并验证留存；待数据与需求验证后，再投入原生 App 与小程序，以获得离线、推送、入口便捷等更强的端能力。

## 2. 用户分析

### 2.1 目标用户

**用户范围**：面向 **18 岁以上**的所有人群（无年龄上限），覆盖学生、职场人、家庭/情侣到中老年用户。

**人群定位（主次）**：
- **主要群体（核心）**：**个人用户**，包括独立记账的个人，以及共同管理日常开支的**家庭 / 情侣**。产品的交互、分类、报表均以这部分用户为第一优先级设计。
- **次要群体（兼顾）**：**小微企业主 / 个体工商户**。他们收支规模不大、无需专业财务软件，但需要简单的收支两条线、公私区分与可导出的流水。本期不做专业财务功能，仅在分类、备注、导出等通用能力上予以兼顾。

| 用户类型 | 归属 | 特征 | 核心诉求 |
| --- | --- | --- | --- |
| 月光族 / 年轻职场人 | 个人（核心） | 收入稳定但存不下钱，消费随意 | 看清钱花在哪，控制冲动消费 |
| 学生 / 初入社会群体 | 个人（核心） | 可支配收入有限，需精打细算 | 低门槛记账、看清小额开支 |
| 家庭 / 情侣记账者 | 个人（核心） | 需共同管理家庭日常开支 | 分类清晰、可按月复盘、可设预算、可共享账本（远期） |
| 中老年 / 储蓄目标人群 | 个人（核心） | 重视存钱与稳健，为旅行、养老等攒钱 | 大字体易用、预算管理、目标进度可视化 |
| 自由职业者 | 个人 / 兼顾经营 | 收支不规律，需区分公私 | 收支两条线、可备注、可导出 |
| 小微企业主 / 个体工商户 | 次要（兼顾） | 经营规模小，无专业财务需求 | 简单收支记录、公私分账、流水导出对账 |

### 2.2 典型用户画像

> **小林，26 岁，互联网公司运营**
> 每月工资到账后没几天就"不知道花哪了"，想攒钱旅行却总是失败。试过几个记账 App，但都因为"记一笔要点好几下、还要选半天分类"而放弃。
> 她的诉求：**打开 App 两三步就能记完，月底能一眼看到这个月哪类花得最多。**

### 2.3 核心使用场景

1. **即时记账**：在便利店买完东西，顺手掏出手机记一笔（金额 + 分类）。
2. **补记账**：晚上回家想起白天还有几笔没记，批量补录。
3. **月度复盘**：月底/月初查看上月消费报表，按分类查看占比与趋势。
4. **预算管控**：月初设定餐饮预算，月中收到"已用 80%"提醒。

## 3. 需求分析

### 3.1 用户痛点

- 记账操作步骤多、太麻烦 → **坚持不下来**。
- 账单分散、手动汇总累 → **看不清全貌**。
- 只有流水没有分析 → **不知道钱花在哪、该省哪里**。
- 没有预算约束 → **花超了也无感知**。

### 3.2 需求优先级（MoSCoW）

| 优先级 | 需求 |
| --- | --- |
| Must（必须有） | 快速记账、收支分类、账单流水、月度统计 |
| Should（应该有） | 预算管理、超额提醒、连续记账激励、数据本地存储与备份 |
| Could（可以有） | 多账本、自定义分类、图表多维分析、数据导出 |
| Won't（本期不做） | 多人协作、银行卡自动同步、AI 智能记账、社区功能 |

## 4. 功能需求

### 4.1 功能结构图

```
个人记账 App
├── 记账模块
│   ├── 快速记账（金额、收/支、分类、备注、日期）
│   ├── 分类选择（系统预置 + 自定义）
│   └── 批量补记
├── 账单模块
│   ├── 流水列表（按日/周/月）
│   ├── 单笔编辑 / 删除
│   └── 搜索与筛选
├── 统计模块
│   ├── 月度收支概览
│   ├── 分类占比（饼图）
│   └── 趋势分析（折线/柱状）
├── 预算模块
│   ├── 月度总预算 / 分类预算
│   └── 预算进度与超额提醒
└── 我的模块
    ├── 账本管理
    ├── 数据备份 / 导出
    └── 设置（提醒、主题、安全锁）
```

### 4.2 核心功能详述

#### 4.2.1 快速记账（P0 核心）

**功能描述**：用户进入 App 后能以最少步骤完成一笔收支记录。

**字段定义**：

| 字段 | 是否必填 | 说明 | 默认值 |
| --- | --- | --- | --- |
| 金额 | 必填 | 数字键盘输入，支持小数点后两位 | 无 |
| 收支类型 | 必填 | 支出 / 收入，默认"支出" | 支出 |
| 分类 | 必填 | 点选分类图标 | 上次使用的分类 |
| 日期 | 必填 | 默认当天，可改 | 当天 |
| 备注 | 选填 | 文本输入，≤ 50 字 | 空 |

**交互要求**：
- 打开 App 默认落在记账页，金额输入框自动聚焦、数字键盘自动弹出。
- 分类以图标网格平铺展示，单击即选中，无需二级菜单。
- "保存"后给出轻量成功反馈（如金额飞入动画 + Toast），并清空表单等待下一笔。
- 单笔记账理想路径：**输入金额 → 点分类 → 保存**，3 步完成，目标耗时 ≤ 10 秒。

**异常处理**：
- 金额为 0 或为空时，"保存"按钮置灰不可点。
- 金额超过单笔上限（如 1,000,000）时给出提示。

#### 4.2.2 收支分类

**功能描述**：提供系统预置分类，并支持用户自定义。

- **预置支出分类**：餐饮、交通、购物、居住、娱乐、医疗、教育、人情、通讯、其他。
- **预置收入分类**：工资、奖金、理财、兼职、红包、其他。
- **自定义分类**：用户可新增分类（图标 + 名称）、隐藏不用的分类、调整排序。
- 删除分类时若该分类下已有账单，需提示"历史账单将归入'其他'"并二次确认。

#### 4.2.3 账单流水

**功能描述**：以时间维度展示所有收支记录。

- 默认按"月"分组展示，顶部显示当月总收入、总支出、结余。
- 列表项展示：分类图标、分类名、备注、日期、金额（支出为红色/收入为绿色）。
- 支持按日/周/月切换视图。
- 单击列表项进入详情，可编辑或删除。
- 支持按关键词（备注）、分类、金额区间、日期范围筛选搜索。

#### 4.2.4 统计分析

**功能描述**：将流水转化为可视化洞察。

- **收支概览**：当月收入、支出、结余的数字卡片，并与上月环比。
- **分类占比**：饼图展示各分类支出占比，点击可下钻查看该分类明细。
- **趋势分析**：折线/柱状图展示近 6 个月或近 30 天的收支趋势。
- 支持切换统计周期（本月、上月、本年、自定义区间）。

#### 4.2.5 预算管理（P1）

**功能描述**：帮助用户设定花费上限并主动管控。

- 支持设置**月度总预算**与**分类预算**。
- 首页/统计页以进度条展示"已用 / 预算"，并按比例变色（绿 → 黄 → 红）。
- 触达阈值时推送提醒：达到 80%、100%、超额时分别提醒。
- 预算可设为每月自动循环。

#### 4.2.6 习惯激励（P1）

**功能描述**：通过正反馈提升记账留存。

- 连续记账天数统计（连续打卡）。
- 每日记账提醒（可设定时间，默认晚 21:00）。
- 里程碑成就（如"连续记账 7 天""累计记账 100 笔"）。

#### 4.2.7 数据安全与备份（P1）

- **本地存储优先**：数据默认存储在本地，保证离线可用。
- **备份/恢复**：支持导出备份文件、从备份恢复；可选云端备份（登录后）。
- **数据导出**：支持导出 CSV / Excel，便于二次分析。
- **隐私安全**：支持手势密码 / 指纹 / 面容锁。

### 4.3 非功能需求

| 类别 | 要求 |
| --- | --- |
| 性能 | 首屏加载 ≤ 2 秒（移动网络下）；记账保存响应 ≤ 300ms；流水列表滚动流畅。 |
| 可用性 | 移动浏览器优先的响应式布局；核心记账流程支持弱网/断网后重连续传（数据先存本地缓存）。 |
| 兼容性 | 移动端：iOS Safari 14+、Android Chrome 90+、微信内置浏览器；PC 端：Chrome / Edge / Safari 主流近两年版本。 |
| 安全性 | 数据传输使用 HTTPS；本地缓存（localStorage/IndexedDB）敏感数据加密。 |
| 可访问性 | 关键操作支持大字体；色弱用户在收支颜色外辅以正负号区分；触控目标尺寸 ≥ 44px。 |

### 4.4 视觉设计规范（粉嫩可爱风）

**设计风格**：整体采用**粉嫩、可爱、温柔治愈**的视觉基调，弱化"理财=严肃数字"的距离感，让记账变得轻松有亲和力。圆角、柔和阴影、留白充足，搭配少量可爱插画/表情元素。

**主色板**：

| 角色 | 颜色名 | 色值（参考） | 用途 |
| --- | --- | --- | --- |
| 主色 | 樱花粉 | `#FF9EC4` | 主按钮、导航高亮、品牌色 |
| 主色浅 | 蜜桃粉 | `#FFD6E7` | 选中态背景、卡片高亮、进度条底 |
| 辅助色 | 奶杏橙 | `#FFC8A2` | 次要按钮、标签点缀 |
| 点缀色 | 薄荷绿 | `#A8E6CF` | 收入正向元素、成就徽章 |
| 背景色 | 奶油白 | `#FFF8FB` | 页面背景（带极浅粉调） |
| 文字主色 | 暖灰 | `#5C5560` | 正文（避免纯黑，更柔和） |

**收支语义色**（兼顾色弱）：
- 支出：玫瑰红 `#FF6F91`，并以"-"前缀强化。
- 收入：薄荷绿 `#3DBE8B`，并以"+"前缀强化。

**风格细节**：
- 圆角统一偏大（卡片 16px、按钮 24px 胶囊形），营造柔软感。
- 阴影用粉调柔光（如 `rgba(255,158,196,0.25)`），不用生硬黑色投影。
- 字体偏圆润（如苹方/思源黑体 Medium，可点缀圆体数字）。
- 适度使用可爱元素：记账成功的小动画、连续打卡的萌系徽章、空状态的治愈插画。
- 保证对比度与可读性：粉色背景上的文字需达到 WCAG AA 对比度，关键数字不因配色牺牲清晰度。

## 5. 业务流程

### 5.1 记账主流程

```
打开 App → 默认进入记账页（金额框聚焦）
  → 输入金额
  → 选择收/支类型（默认支出）
  → 点选分类
  → （可选）填备注 / 改日期
  → 点击保存
  → 校验金额是否合法
      ├── 合法 → 写入本地数据库 → 成功反馈 → 清空表单
      └── 不合法 → 提示错误，停留当前页
```

### 5.2 预算提醒流程

```
用户设定分类预算
  → 每笔记账后实时累计该分类当月支出
  → 计算 已用/预算 比例
      ├── < 80%  → 仅更新进度条
      ├── ≥ 80%  → 推送"接近预算"提醒
      └── ≥ 100% → 推送"已超预算"提醒（高亮红色）
```

## 6. 数据指标（成功度量）

| 指标类型 | 指标 | 目标 |
| --- | --- | --- |
| 获取 | 新增用户数 | 按运营目标设定 |
| 激活 | 首日完成首笔记账率 | ≥ 60% |
| 留存 | 次日留存 / 7 日留存 | 次留 ≥ 40% |
| 核心行为 | 人均日记账笔数 | ≥ 2 笔 |
| 习惯 | 连续记账 ≥ 7 天用户占比 | ≥ 20% |
| 效率 | 单笔记账平均耗时 | ≤ 10 秒 |
